# SpringBoot Architecture Lab

## 📘 Overview
The **SpringBoot Architecture Lab** is a comprehensive repository designed to explore, experiment, and document modern software architecture practices using Spring Boot.  
It serves as a **lab environment** for architects, developers, and product owners to understand and implement solutions across multiple domains — cloud platforms, design patterns, reactive programming, and event-driven systems.

---

## 🎯 Objectives
- Provide **hands-on examples** of Spring Boot integrations with AWS, GCP, and Azure.
- Demonstrate **design patterns** and their application in real-world scenarios.
- Showcase **reactive programming** using Project Reactor and Spring WebFlux.
- Implement **scenario-based architectures** for microservices, APIs, and distributed systems.
- Explore **event-driven patterns** with Kafka, RabbitMQ, and cloud-native event services.
- Document **best practices** for scalability, security, and maintainability.

---

## 🧩 Features

### ☁️ Cloud Integrations
- AWS (S3, Lambda, DynamoDB, SNS/SQS)
- GCP (Pub/Sub, Cloud Storage, Firestore)
- Azure (Blob Storage, Event Hub, Cosmos DB)

### 🏗️ Design Patterns
- Creational: Singleton, Factory, Builder
- Structural: Adapter, Composite, Proxy
- Behavioral: Observer, Strategy, Command
- Scenario-based: API Gateway, Circuit Breaker, CQRS

### ⚡ Reactive Java
- Spring WebFlux
- Project Reactor
- Reactive Streams with MongoDB and R2DBC

### 📡 Event-driven Architectures
- Kafka producers/consumers
- RabbitMQ messaging
- Cloud-native event services (SNS, Pub/Sub, Event Hub)
